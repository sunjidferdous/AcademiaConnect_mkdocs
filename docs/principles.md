# UI Design Principles

## Mandel's Golden Rules Applied

### 1. Place Users in Control

- **Modeless**: No modal dialogs unless necessary
- **Flexible**: Keyboard and mouse/touch support
- **Interruptible**: Can cancel any action
- **Helpful**: Clear error messages and guidance
- **Forgiving**: Undo/redo where applicable
- **Navigable**: Breadcrumbs and clear navigation
- **Accessible**: Works for various skill levels
- **Facilitative**: UI doesn't get in the way
- **Preferences**: Customizable settings
- **Interactive**: Direct manipulation of objects

### 2. Reduce Users' Memory Load

- **Remember**: System remembers filter preferences
- **Recognition**: Icons and labels, not just icons
- **Inform**: Visual cues for state and context
- **Forgiving**: Provide defaults and undo
- **Frequency**: Shortcuts for common actions
- **Intuitive**: Object-action syntax (select, then act)
- **Transfer**: Use familiar metaphors (inbox, feed)
- **Context**: Progressive disclosure of options
- **Organize**: Clear visual hierarchy

### 3. Make the Interface Consistent

- **Continuity**: Maintain user task context
- **Experience**: Consistent within and across sections
- **Expectations**: Predictable interaction results
- **Attitude**: Aesthetic appeal and integrity
- **Predict**: Encourage exploration safely

---

## Additional Design Principles

### Mobile-First Approach
- Responsive breakpoints
- Touch-friendly targets (min 44x44px)
- Optimize for one-hand use

### Accessibility
- WCAG 2.1 AA target
- Color contrast ratios
- Screen reader support
- Keyboard navigation

### Performance
- Lazy loading for images
- Infinite scroll for feeds
- Optimistic UI updates
- Skeleton screens while loading
