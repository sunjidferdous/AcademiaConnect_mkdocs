# Glossary

## A

**Agile**: Iterative software development methodology emphasizing flexibility and collaboration

**API (Application Programming Interface)**: Set of protocols for building and integrating software applications

**Authentication**: Process of verifying user identity

**Authorization**: Process of determining user permissions and access levels

---

## C

**CDN (Content Delivery Network)**: Distributed network of servers that delivers web content based on geographic location

**CSRF (Cross-Site Request Forgery)**: Security vulnerability where unauthorized commands are transmitted from a user that the web application trusts

---

## E

**ELMS (Electronic Learning Management System)**: Software platform for managing educational courses and content

**E2E Testing (End-to-End Testing)**: Testing methodology that validates entire application flow from start to finish

---

## G

**GDPR (General Data Protection Regulation)**: European Union regulation on data protection and privacy

---

## J

**JWT (JSON Web Token)**: Compact, URL-safe means of representing claims to be transferred between two parties

---

## L

**LDAP (Lightweight Directory Access Protocol)**: Protocol for accessing and maintaining distributed directory information services

**LMS (Learning Management System)**: See ELMS

---

## M

**MVP (Minimum Viable Product)**: Version of a product with just enough features to be usable by early customers

---

## O

**OAuth2**: Authorization framework enabling applications to obtain limited access to user accounts

**OSS (Open Source Software)**: Software with source code that anyone can inspect, modify, and enhance

---

## P

**PWA (Progressive Web App)**: Web application that uses modern web capabilities to deliver app-like experience

---

## R

**RBAC (Role-Based Access Control)**: Method of regulating access based on roles of individual users

**REST (Representational State Transfer)**: Architectural style for designing networked applications

---

## S

**SAML (Security Assertion Markup Language)**: Open standard for exchanging authentication and authorization data

**SRS (Software Requirements Specification)**: Complete description of the behavior of a system to be developed

**SSO (Single Sign-On)**: Authentication scheme allowing user to log in with single set of credentials to multiple systems

---

## T

**TLS (Transport Layer Security)**: Cryptographic protocol for secure communications over computer network

---

## U

**UCAM**: University Course and Attendance Management system (specific to UIU)

**UI/UX (User Interface/User Experience)**: Combined field of design focused on appearance and experience of products

---

## W

**WebSocket**: Communication protocol providing full-duplex communication channels over single TCP connection

**WCAG (Web Content Accessibility Guidelines)**: Accessibility guidelines for web content

---

## X

**XSS (Cross-Site Scripting)**: Security vulnerability allowing attackers to inject client-side scripts into web pages
