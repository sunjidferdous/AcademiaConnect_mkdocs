## Change Request Process

1. **Identify Need**: Stakeholder or team member identifies required change
2. **Document**: Create change request with justification
3. **Review**: Team reviews impact on scope, timeline, and resources
4. **Approve**: Supervisor or project lead approves/rejects
5. **Implement**: If approved, update documentation
6. **Log**: Record in this Change Log
